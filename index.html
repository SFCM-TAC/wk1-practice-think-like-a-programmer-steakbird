Think Like a Programmer

Pretend you are writing a plugin that automatically harmonizes with MIDI input. What instructions would you give a computer to do this programmatically?

  • 1. Identify a range of notes
  • 2. Identify different
  • 3. Quantization/scale settings
  • 4. Define harmonization rules for different quantization settings
  • 5. Listen for midi note input
  • 6. When a midi note is played, determine which note that is
  • 7. Refer to scale and quantization setting and determine where the played note falls onto the scale
  • 8. Play next 3 harmonically correct notes depending on quantization settings

    • Pick an instrument and pretend you are writing an app that mimics that instrument. How would you model the istrument in a way the computer would understand? Think about what different objects (e.g. "string" for a guitar) it might need and what properties (e.g. "pitch"), events (e.g. "pluck"), and methods (e.g. "playPitch()") would be useful.

    A guitar can be broken down into a few corresponding objects. Strings can be thought of as separate objects, but grouped together in the same class (they should be given different id parameters though). Tuning pegs can also be objects that are acted upon to change tuning. In our string example, the string can have various properties like material, thickness, or brand. An event occurs when something triggers an action with an object. An event might be when the string is plucked, when a finger is pressed on the fretboard, or even when the guitar is slammed against the ground. A method as far as I know is a function that is run to produce a result. So the event stringPluck would call the method playPitch() to occur whenever it happened, and similarly an event called retuning could call the method reduceTension() on the tuning peg object.
